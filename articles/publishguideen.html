Azure Marketplace Service Provider Guide [中文版]

This guide provides ISV overall procedures from entering Azure Marketplace to product release.

Scope

  • This guide is intended for ISVs that wish to provide virtual machine (hereinafter refer to as VM) image products, ARM templates and custom services (i.e. solutions) for Chinese users in Azure Marketplace by Microsoft Azure, which are operated by 21Vianet.
  • This guide does not apply to service providers (such as SaaS providers, API service providers, etc.) offering non-VM images or custom service offerings.

What is Azure Marketplace

Azure Marketplace is an image repository for third-party ISVs to publish their image and custom services; and to help Azure users quickly create VM instances, clusters or find suitable Azure-based solutions, which helps connect Azure users, Azure ISVs or partner directly.

Azure Marketplace makes it easier to find, retrieve, and use well-tested and fully tested applications on the Azure platform and makes sure the application running smoothly on Azure.

ISVs, through Azure Marketplace, can bring new business channels and users; and dramatically simplify the software promotion, management and deployment.

Product

Azure Marketplace currently supports VM Image, ARM Template, and Custom Services. More products will be supported in the future.

Business model

Azure Marketplace currently supports free and Bring Your Own License (BYOL) model.

The following table describes business models currently supported in Azure Marketplace. A product / service can include SKUs that use different pricing models.

Pricing model Description Object
Free Free SKU. No charge for Azure Marketplace and the product / service. VM image, ARM template
Free Trial SKUs are free in limited time. During the trial period, Azure Marketplace fees will not be charged for usage of the product. After the trial expires, customers using the product are automatically charged based on the standard rates. Azure Marketplace does not currently prevent users from creating multiple subscriptions for free trial products. Service providers that create free trial product subscriptions for customers are expected to provide the appropriate restrictions. VM image, ARM template
BYOL Comes with a license (BYOL) SKU. Customer is entitled to access or use the Product / Service outside of the Azure Marketplace and does not charge the customer for the Azure Marketplace fee due to Customer's usage of the Product / Service in it. VM image, ARM template

Azure Marketplace Release Process

1. Become an Azure Marketplace service provider

Before registration, please confirm the requirements to service providers in Azure Marketplace. Please refer to Azure Marketplace Participation Policy.

  • Submit an online application to register as an Azure Marketplace Provider: Vendor Registration
  • Tip: Make sure you have such an Azure account before submitting your registration information: someone@example.partner.onmschina.cn; if you do not have it, you can apply directly: Azure Trial Application Form (Please select "Enterprise Application" on registration)
Prompt:

Note: You must register with a legally operating entity with a registered address in PRC.

  • Azure team will review the application and the results will be sent to your designated email address to contact you

  • After obtaining the service provider account, the service provider can log in to Azure Marketplace, browsing Azure Marketplace content and related technical documents.

2. Build a VM image, an ARM template or a custom service for deployment

  • For a detailed process of creating a VM image, please refer to: Azure Marketplace Virtual Machine Image Creation Guide
  • For the process of making an ARM template, please refer to: Creating an ARM Template
  • For custom services, solution-related documents and other resources need to be prepared.
  • After the image is created and tested, it needs to be uploaded to a public Azure storage account. The ARM template file can be stored in an Azure storage account or a non-Azure storage account
  • You can contact Azure Marketplace Technical Support

3. Pre-release

Pre-release refers to simulating actual product launch in a "sandbox" environment. Make sure you have obtained the service provider account before posting. You can refer to Azure Marketplace VM Image Release Guide and other release documents.

  • Log in to the publishing portal using your service provider account
  • Fill in the release application form, submit an image, ARM template file or custom service before applying for release
  • After approval, Azure Marketplace team informs the service provider of pre-online testing and verification by email
  • After finishing testing, the service provider needs to inform the Azure marketing team of the test results for confirmation

4. Release

  • After the service provider passes the test, Azure Marketplace team will double confirm your product and publish it to Azure Marketplace
  • After the announcement is completed, you will be notified by email that your product has been officially released
  • You can view your published images, ARM templates, or custom services in Azure Marketplace
